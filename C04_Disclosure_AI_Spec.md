# C04 — AI Disclosure Components Specification
## Art. 50 EU AI Act — Transparency Obligations

**Versione:** 1.0  
**Data:** 2026-05-21  
**Scope:** Tutti i componenti UI/output generati da AIComply  
**Norma di riferimento:** Regolamento UE 2024/1689 Art. 50 (Obblighi di trasparenza per determinati sistemi di IA)

---

## 1. Contesto normativo

L'Art. 50 del Regolamento UE 2024/1689 impone obblighi di trasparenza per i sistemi AI che interagiscono con persone fisiche o generano contenuti. AIComply, in quanto piattaforma che genera documentazione di compliance assistita da AI, rientra nell'ambito applicativo dell'articolo.

**Obblighi rilevanti:**
- Art. 50(1): Gli utenti devono essere informati che stanno interagendo con un sistema AI
- Art. 50(3): I contenuti generati da AI devono essere marcati come tali
- Art. 50(4): I provider devono garantire che gli output AI siano riconoscibili come tali

### 2.3 Label Output AI — Header di ogni card/blocco generato

Ogni card o sezione che contiene output generato o analizzato da AI deve avere un header strutturato come segue.

#### Struttura visiva (da sinistra a destra, su due righe)

**Riga 1 (badge + tipo + badge):**
```
[🤖 Generato da AI]   [Tipo: {documentType}]   [⚖ Richiede revisione legale]
```

**Riga 2 (metadati):**
```
{data} {ora} · Modello: {modelName} · ID: {outputId} · v{systemVersion}
```

#### Campi obbligatori

| Campo | Descrizione | Formato |
|-------|-------------|---------|
| `documentType` | Tipo documento (es. "Documentazione Tecnica Art. 11", "Valutazione del Rischio Art. 9") | String libera |
| `modelName` | Nome modello AI letto dalla config (non hardcoded) | es. "Gemini 1.5 Pro" |
| `outputId` | ID univoco output | `[TIPO]-[YYYYMMDD]-[NNN]` |
| `systemVersion` | Versione piattaforma AIComply | da env/package.json |
| `generatedAt` | Timestamp generazione | ISO 8601, visualizzato come `DD/MM/YYYY HH:mm` |

#### Formato ID output

Pattern: `[TIPO]-[YYYYMMDD]-[NNN]`

| Tipo documento | Codice TIPO |
|----------------|------------|
| AIA-Architect (Art. 11) | `AIA` |
| Risk Manager (Art. 9) | `RSK` |
| Data Audit (Art. 10) | `DAT` |
| DocuGen | `DOC` |
| LogVault | `LOG` |
| Transparency (Art. 13) | `TRS` |
| Oversight (Art. 14) | `OVS` |
| Resilience (Art. 15) | `RES` |
| QMS (Art. 17) | `QMS` |
| FRIA (Art. 49) | `FRI` |
| Conformity (Art. 43) | `CNF` |
| GPAI (Art. 51-55) | `GPA` |
| XAI (Art. 13+) | `XAI` |
| Dossier completo | `DOS` |
| Generico | `GEN` |

`[NNN]` = contatore sequenziale per sessione, a 3 cifre (001, 002, ...), resettato a 001 ogni giorno.

**Il generatore di ID deve essere condiviso con il sistema di audit trail (C-03) tramite un modulo centralizzato `@/lib/disclosure/output-id`.**

#### Tag [verify-pinpoint] per citazioni normative

Ogni citazione di articoli normativi nell'output AI deve essere seguita da un badge cliccabile:

```
Art. 9 del Regolamento UE 2024/1689 [▲ verifica]
```

**Stile badge:**
```
background: #FEF3C7 (amber-50)
color:       #92400E (amber-800)
border:      1px solid #FDE68A (amber-200)
font-size:   10px
cursor:      pointer
```

**Al click:** apre un tooltip o side panel con:
- Testo ufficiale dell'articolo citato (o link alla fonte ufficiale EUR-Lex)
- URL: `https://eur-lex.europa.eu/legal-content/IT/TXT/?uri=CELEX%3A32024R1689`

**Implementazione:** il componente `<VerifyPinpoint article="Art. 9" />` wrappa il testo della citazione e aggiunge il badge.

## 3. Generatore ID centralizzato

File: `src/lib/disclosure/output-id.ts`

```typescript
// Genera ID univoci condivisi con audit trail (C-03)
// Formato: [TIPO]-[YYYYMMDD]-[NNN]

export type OutputType =
  | "AIA" | "RSK" | "DAT" | "DOC" | "LOG" | "TRS"
  | "OVS" | "RES" | "QMS" | "FRI" | "CNF" | "GPA"
  | "XAI" | "DOS" | "GEN";

export function generateOutputId(type: OutputType): string {
  const today = new Date();
  const date = today.toISOString().slice(0, 10).replace(/-/g, "");
  const counterKey = `aicomply_output_counter_${date}`;
  const current = parseInt(localStorage.getItem(counterKey) ?? "0") + 1;
  localStorage.setItem(counterKey, String(current));
  const nn = String(current).padStart(3, "0");
  return `${type}-${date}-${nn}`;
}
```

## 4. Config modello AI

Il nome del modello va letto da:
1. `process.env.NEXT_PUBLIC_AI_MODEL_NAME` (runtime config)
